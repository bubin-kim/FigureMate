# M1.5 — LLM Provider Layer

> **문서 이력**: 이 문서는 구현(2026-07-12, M2 완료 직후) 후 사용자 요청으로 작성된
> 사후 기록이다. 다른 마일스톤 문서(M1/M2)와 달리 "작업 지시서"가 아니라 "실제 내린
> 설계 결정의 기록"이며, 구현 당시 채팅으로 받은 요구사항을 기준으로 한다.
> 번호가 M1.5인 이유: 특정 파이프라인 단계가 아니라 M3~M4가 공유할 인프라 계층이기 때문.

## 목표

FigureMate가 특정 LLM에 종속되지 않도록 Provider 추상화 계층을 도입한다.
환경변수만 변경하면 Mock → Ollama → Anthropic → (OpenAI)로 전환 가능해야 하며,
**개발 중에는 API 비용이 발생하지 않아야 한다.**

## 요구사항 (구현 당시 지시)

- 기본 Provider는 MockProvider — 실제 API를 호출하지 않고 개발용 더미 데이터 반환
- OllamaProvider를 두 번째 Provider로 구현 (로컬, 무비용)
- AnthropicProvider는 Optional Provider (키 필수, 유료)
- OpenAI까지 환경변수로 자유 전환 가능한 설계

## 구현 결과

```
backend/app/core/llm/
├── __init__.py            # get_llm_provider()/get_embedding_provider() 팩토리 + 레지스트리
├── base.py                # LLMProvider/EmbeddingProvider 인터페이스, Message 타입
├── mock_provider.py       # 기본. 결정론적 더미 (complete + embed 768차원)
├── ollama_provider.py     # 로컬 HTTP (complete + embed)
└── anthropic_provider.py  # Optional (complete + vision, 생성 시점 키 검증)
```

환경변수: `LLM_PROVIDER=mock|ollama|anthropic|openai(슬롯)`, `EMBEDDING_PROVIDER=mock|ollama`
(상세는 `docs/architecture.md` 4.5절, `.env.example` 참고)

## 주요 설계 결정과 근거

1. **위치: `app/services/llm/`이 아니라 `app/core/llm/`.**
   CLAUDE.md의 폴더 의미론상 services는 "PDF 파싱 등 결정론적 로직 (Agent 아님)"이고,
   LLM 호출은 비결정론적이며 Provider는 특정 파이프라인 단계가 아니라 여러 Agent가
   공유하는 횡단 인프라다 (설정·DB 세션과 같은 계열). 개발 원칙 5번("LLM 호출과
   결정론적 로직을 구분한다")의 구분을 폴더 구조에도 그대로 반영한 것.

2. **LLM/임베딩 Provider 설정 분리 (`LLM_PROVIDER` + `EMBEDDING_PROVIDER`).**
   Anthropic은 임베딩 API가 없으므로(Claude는 completion/vision 전용) 단일 스위치면
   "환경변수 하나로 전환" 약속이 Anthropic 선택 시 깨진다. 분리하면 LLM=anthropic +
   임베딩=ollama 같은 조합이 가능하다. (사용자 승인 완료)

3. **OpenAIProvider는 최소 구현 대신 "레지스트리 슬롯 예약 + 안내 에러".**
   키 없이는 E2E 검증이 불가능하므로 최소 구현은 "테스트 없는 죽은 코드"가 되어
   이 프로젝트의 관례(모든 코드는 테스트와 함께, DoD로 검증)를 어기는 유일한 예외가
   된다. 또한 사용하지 않는 `openai` SDK 의존성이 추가되고, 검증 없이 호출 준비된
   유료 API 코드가 존재하는 것은 비용 가드레일 취지와 어긋난다. 슬롯 방식은
   `LLM_PROVIDER=openai`가 인식되고 추가 절차(`__init__.py` docstring 4단계)를
   안내하는 에러를 던져 전환 계약은 유지한다. 실제 구현은 필요 시점에 테스트와 함께.
   (사용자 동의 완료)

4. **비용 가드레일**: 기본값 mock(무설정 = 유료 호출 0, 테스트로 강제),
   AnthropicProvider는 생성 시점 키 검증(조용한 실패 없음), 테스트 14개 전부 네트워크 0
   (Ollama는 httpx.MockTransport). `docs/roadmap.md` 비용 절에 반영됨 — M3~M4를
   mock/ollama로 개발하는 동안 비용이 발생하지 않는다.
