# 로드맵

`docs/milestones/`의 M1~M5가 현재 실행 대상입니다. 이 문서는 그 이후의 방향성을 짧게
기록해두는 곳으로, 아직 상세 설계 문서는 아닙니다. M5가 끝나면 이 목록을 근거로 다음
마일스톤 문서를 작성합니다.

## ⚠️ 비용 발생 지점 — M4 이전에 반드시 확인

M1~M2는 Anthropic API 키 없이 완전 무료로 진행 가능합니다 (PyMuPDF + 정규식/휴리스틱).
M3부터는 임베딩/LLM 검증이 부분적으로 필요할 수 있고, **M4(Figure 설명 생성)는 Vision
모델 호출이 핵심이라 API 키 없이는 구현할 수 없습니다.**

M3 착수 전, M4 착수 전에는 각각:
1. Anthropic API 키를 발급하고 사용량 기반 과금에 동의했는지
2. (대안으로) 로컬 오픈소스 Vision 모델로 대체할지, 아니면 해당 마일스톤을 보류할지

를 **사용자에게 먼저 확인**합니다. Claude Code는 API 키가 없는 상태에서 임의로 유료 API
호출 코드를 작성하거나 실행하지 않습니다. 확인 없이 진행하지 마세요.

**갱신 (2026-07-12, LLM Provider Layer 도입)**: `backend/app/core/llm/`에 Provider 추상화가
추가되어 위 2번 대안이 실제 구조로 구현되었습니다. `LLM_PROVIDER`/`EMBEDDING_PROVIDER`
환경변수로 mock(기본, 비용 0) → ollama(로컬, 비용 0) → anthropic(유료, 키 필수) 전환이
가능하며, **M3~M4를 mock/ollama로 개발하는 동안에는 API 비용이 발생하지 않습니다.**
Anthropic(또는 향후 OpenAI)으로 실제 전환하는 시점에만 위 확인 절차를 거치면 됩니다
(architecture.md 4.5절 참고).

## M1~M5 (현재 범위)

파이프라인 5단계: 업로드 → PDF 분석 → Figure 추출 → Figure↔본문 매칭 → 설명 생성 → 최소 뷰어 UI.
상세는 `README.md`의 개발 순서와 `docs/milestones/M1.md` 참고.

## M6 이후 (방향성만, 우선순위 미확정)

- **대화형 Q&A**: Figure에 대해 사용자가 자유 질문 → 매칭된 본문 근거로 답변
- **발표 스크립트 생성**: 선택한 Figure들을 중심으로 N분 발표 대본 생성
- **예상 질문 생성**: Figure별로 나올 법한 질문과 모범 답변 초안
- **난이도 조절**: 같은 설명을 학부/석사/전문가 눈높이로 재생성
- **수식(Equation) 지원**: Figure와 마찬가지로 수식을 추출하고 단계별로 설명
- **여러 논문 간 비교**: 같은 개념이 다른 논문에서 어떻게 다르게 쓰였는지 비교

이 항목들은 우선순위나 상세 설계가 확정되지 않았습니다. 각 항목을 실제로 진행할 때
`docs/adding-an-agent.md` 절차에 따라 새 마일스톤 문서를 만들고 착수합니다.
